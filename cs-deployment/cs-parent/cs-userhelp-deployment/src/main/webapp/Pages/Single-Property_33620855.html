Sirma Platform : Single Property

The user is able to insert a single property of a selected object in an iDoc using the "Single Property" widget. The property  could be inserted in the text the user is writing.

The user is able to insert a property even if it is empty in order to input value later or to edit the existing property from the widget.

Rules:

  1. The widget could be inserted in any place in an idoc - in the beginning of a row, at the end of the row, in the middle of a text, etc.
  2. More than one widgets of this type could be inserted in one row.
  3. The widget allows to search for an object and select the property that to be displayed in it.
  4. The widget allows the user to select if the property to be displayed in preview mode of the document in a border (as a cell) or as simple text.
  5. The widget changes based on the selected property type - if the property is date format - then the widget changes to a field with calendar; if the property is selected from a codelist - then the widget changes to a dropdown box, in which the user could select value from the respective codelist; if the property is a text field - the the widget is free text field.
  6. The widget can be resized manually.
  7. If the text in the widget is longer than the widget width, then the widget is automatically resized in hight - the text in it is displayed in more than 1 rows.
  8. If the selected property to be displayed in the widget is empty (there is no value) then the widget allows the user to enter value in it, when the iDoc is in edit mode. The property is saved for the selected object.
  9. The user could edit a property displayed in the widget when the iDoc is in edit mode.
  10. If a property displayed in the widget is read-only (not allowed for editing), then the widget does not allow the user to edit it.
  11. The font type and font size in the widget are the same as in the whole document.
  12. Deleting the widget does not delete the property.

Insert a property with value in iDoc

  • The user opens an iDoc in edit mode, inserts text in it (1) and inserts the single property widget where necessary (2-3).
  • The user selects "Manage Properties" (4-5).

Note: In the example the Object Data widget is inserted in the iDoc to display some properties of the document which are used in the Single Property widget.

  • The system opens the "Single Property" pop-up.
    • By default the tab "Object to be displayed" is selected (1)
    • By default the option "Display information for the current object" (2) is selected and the current document (3) is displayed (if saved at least once). The user may select the other options "Automatically select from current context" or "Manually select object' and choose the object to display a selected property.
  • The user selects tab "Object details to be displayed" (4).



  • The system displays the properties of the selected object.
    • The user selects the value to be displayed in border (1) when the iDoc is in preview mode.
    • The user selects property "Type" (2) to be displayed in the widget
  • The user clicks "OK" (3).

  • The selected property is displayed in the widget (1). The user may extend the widget (2).

  • When the user is ready (1) he/ she saves the document (2-3).

  • The system saves the changes and displays the document in preview mode. The selected single property is displayed in border in the document's content.


Insert empty object property in iDoc and enter text in it

  • The user opens an iDoc in edit mode, inserts text in it, inserts the single property widget where necessary (2) and selects the property "Description" which is empty for that object. The property is free text field (1).
  • The user enters text in the empty field (3) and resizes the widget if necessary. The widget is automatically expanded when the entered text becomes longer than 1 line.
  • The user saves the iDoc (4-5).

  • The iDoc is saved. The entered text is displayed in the widget (1) and is stored in the property "Description" (2) of the object (here - the iDoc).
    • If the user has selected not to show borders - in preview mode the entered text would be displayed as simple text, invisible to be in a widget for the reviewers (without borders).

Insert empty object property in iDoc and select value in it from a code list

  • The user opens an iDoc in edit mode, inserts text in it (1), inserts the single property widget where necessary (2) and selects to display the property "Field with CL999" which is empty for that object and there is a code list of selectable values applied for that field. 
  • The user clicks in the empty field and selects a value from the drop-down menu in (3-4-5) and resizes the widget if necessary. The widget is automatically expanded when the entered text becomes longer than 1 line.

  • The user saves the iDoc and the selected value from the code-list becomes a property of the object (in this example - the current iDoc).



Insert empty object property in iDoc and select date from calendar

If the selected property to be displayed is an empty field where only dates may be entered, the user is offered to select the date from a calendar.
